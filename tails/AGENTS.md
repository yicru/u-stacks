# TAILS STACK

Next.js 15 App Router + Radix UI (shadcn/ui) + Better Auth + Drizzle ORM + Cloudflare Pages.

## STRUCTURE

```
tails/
├── src/
│   ├── app/                    # Next.js App Router pages
│   │   └── api/[[...route]]/   # Hono API catch-all handler
│   ├── components/ui/          # shadcn/ui (auto-managed, Biome-excluded)
│   ├── features/               # Domain logic (auth/, tasks/)
│   ├── hooks/                  # Custom React hooks
│   ├── lib/                    # Client utils (api client, theme, cn)
│   └── providers/              # Context providers (TanStack Query)
├── server/
│   ├── db/                     # Drizzle schema + client
│   ├── lib/                    # Auth setup, DB connection, auth utils
│   ├── routes/                 # Hono handlers (auth.ts, task.ts)
│   └── types/                  # Server type definitions
├── drizzle.config.ts           # Drizzle Kit (Turso/libSQL)
├── open-next.config.ts         # OpenNext Cloudflare adapter
└── wrangler.jsonc              # Cloudflare Workers bindings
```

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Add page | `src/app/` | Next.js App Router convention |
| Add API endpoint | `server/routes/` | Hono route → register in `server/index.ts` |
| Add UI component | `src/components/ui/` | Use shadcn/ui CLI |
| Auth config | `server/lib/auth.ts` | Better Auth setup |
| DB schema | `server/db/schema.ts` | Hand-written Drizzle (SQLite dialect) |
| Auth schema | `server/db/auth-schema.ts` | Auto-generated — do NOT edit |
| Add feature | `src/features/<name>/` | Components + hooks per domain |
| Env vars | `.env.example` | `TURSO_*`, `BETTER_AUTH_SECRET`, `NEXT_PUBLIC_APP_URL` |
| Deploy | `npm run deploy:production` | OpenNext → Cloudflare Pages via dotenvx |
| Server utils | `server/lib/` | errors.ts (5 error classes), auth.ts, auth-utils.ts, api-helpers.ts, id.ts (nanoid) |

## CONVENTIONS

- Path alias: `@/*` → `src/*`, `@server/*` → `server/*`
- Hono factory injects `db` and `requireAuth` into context via middleware
- `src/components/ui/` is shadcn/ui managed — Biome excludes this directory
- `cloudflare-env.d.ts` auto-generated by `npm run cf-typegen`
- Cloudflare bindings typed via `Env.Bindings: CloudflareEnv` in factory
- Biome v2.1.4 (newer than other stacks)
- TanStack Query v5 for server state (`useSuspenseQuery` + `apiClient`)
- Tailwind CSS v4 with `@theme inline` in `globals.css`
- `src/lib/api-client.ts` exports `apiClient` (Hono RPC) + `parseResponse` helper
- `src/lib/auth-client.ts` exports Better Auth client (`useSession` hook)
- `src/lib/env.ts` — Zod-validated environment variables
- `server/lib/errors.ts` — HTTPException-based error classes: BadRequest, Unauthorized, Forbidden, NotFound, InternalServer
- `server/lib/id.ts` — nanoid(14) with custom alphabet for ID generation
- `server/lib/api-helpers.ts` — `logError()` (dev-only) + `createErrorResponse()` JSON helper
- Deploy uses dotenvx for env-specific builds (`dotenvx run -f .env.production --`)

## COMMANDS

```bash
npm run db:migrate            # Push Drizzle schema to Turso
npm run better-auth:generate  # Regenerate auth schema
npm run cf-typegen            # Generate Cloudflare env types
npm run deploy:staging        # Deploy staging (dotenvx)
npm run deploy:production     # Deploy production (dotenvx)
npm run preview               # Local Cloudflare preview
```
